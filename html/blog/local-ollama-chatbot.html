Chatting with your Local Ollama

How to use this

If you're running Ollama locally, you have a powerful LLM API running right on your machine. Because your browser is also running locally, this webpage can talk directly to your Ollama instance without sending any of your data to my server!

Step 1: Install and Start Ollama

Make sure you have Ollama installed and running. By default, web browsers block web pages from making requests to local servers for security reasons (CORS). To allow this webpage to talk to your local Ollama, you must start Ollama with the OLLAMA_ORIGINS environment variable set to allow all origins.

Note: Your local host is usually http://localhost:11434 or http://127.0.0.1:11434 depending on your system setup.

Windows (Command Prompt):
set OLLAMA_ORIGINS="*" && ollama serve

Mac/Linux (Terminal):
OLLAMA_ORIGINS="*" ollama serve

Step 2: Connect

Once Ollama is running with CORS enabled, you can use the "Fetch Available Models" button below to load your installed models, or manually type the model name (e.g., llama3) and start chatting!
