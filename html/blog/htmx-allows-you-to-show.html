Let me send you a component - HTMX

Unlike the most popular paradigm in frontend development devoloped in the last few years, single page applications, HTMX is based on Multi page applications. This essentially means that the frontend is not only constantly talking to the backend, but the content that it gets is not just the specific information that we need for the page, It's the parts of the page itself that are transferred over.


In most modern frontend frameworks, you set up your components as primary pieces of code, which exist as compiled files on the Client side. For example, in React, you might have a component called `Button.js`. This component was most likely compiled before the client ever touched the frontend. This both limits and speeds up the development process.


However, with HTMX, you're primary role is to send these components from the backend, where code is much more easily hidden. You can of course get similar effects from single page applications, but the way these frameworks are set up limits you from this front-end approach.


This is not a new idea, but it's interesting nonetheless, especially when you consider all the new back-end capabilities that have become avaliable in the last few years. So, this old type approach deserves to be explored.
